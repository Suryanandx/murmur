# Security Policy

## Supported versions

| Version | Supported |
|---|:---:|
| 0.1.x | ✓ |

## Reporting a vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Use [GitHub Security Advisories](https://github.com/murmur-voice/murmur/security/advisories/new) to report privately. We will acknowledge within 72 hours and aim to patch within 14 days.

## Scope

- API key leakage or exposure in logs/output
- Audio data exfiltration without user consent
- Code injection via config file parsing
- Dependency vulnerabilities (report via Dependabot or directly)

## Out of scope

- Issues requiring physical access to the user's machine
- Social engineering attacks
